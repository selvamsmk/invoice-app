# AGENTS.md

This file documents how to work in this repo for agentic tools.

## Repo overview
- Monorepo managed by Turborepo.
- Runtime/tooling: Bun, TypeScript, Biome.
- Apps: web (Vite + React), server (Elysia + oRPC), desktop (Tauri + Vite).
- Packages: api, auth, app-ui, db, config.

## Build, lint, and test commands
Run commands from repo root unless noted.

### Install
- `bun install`

### Development
- `bun run dev` (web + server via turbo)
- `bun run dev:web`
- `bun run dev:server`
- `bun run dev:native` (desktop)
- `bun run dev:fullstack` (server hot run)

### Build
- `bun run build` (all packages/apps)
- `bun run build:desktop`
- `bun run build:desktop:mac`
- `bun run tauri:build:desktop`

### Lint / format
- `bun run check` (Biome lint + format with fixes)

### Typecheck
- `bun run check-types` (turbo)
- `bun run --cwd apps/web check-types`
- `bun run --cwd apps/server check-types`

### Database (Drizzle + Turso/SQLite)

- `bun run db:migrate`
- `bun run db:generate`
- `bun run db:studio`
- `bun run --cwd packages/db db:local` (start local DB)

## Coding standards
These conventions are inferred from repo config and current code.

### Formatting
- Use Biome (`bun run check`) for formatting and linting.
- Indentation: tabs (Biome `indentStyle: tab`).
- Quotes: double quotes in JavaScript/TypeScript.
- Organize imports: Biome auto-organizes imports.
- Tailwind class sorting: Biome warns on unsorted classes for `clsx`, `cva`, `cn`.

### Imports
- Prefer type-only imports when used for types: `import type { X } from "..."`.
- Keep external imports before internal/workspace imports.
- Use `@` alias for web app source (`apps/web` Vite config).
- Desktop app uses `@` to point at `packages/app-ui/src`.

### TypeScript
- Strict mode enabled across packages.
- `noUncheckedIndexedAccess`, `noUnusedLocals`, `noUnusedParameters` are on.
- Use inferred types where possible; avoid redundant type annotations.
- Prefer `type` over `interface` for simple shapes unless extending/merging.
- Use `as const` when narrowing literals; Biome enforces `useAsConstAssertion`.

### Naming conventions
- Files: `kebab-case` or `camelCase` depending on existing folder conventions.
- React components: `PascalCase` file and symbol names.
- Hooks: `useX` naming.
- Constants: `camelCase` or `SCREAMING_SNAKE_CASE` based on scope and usage.
- Database schema uses `camelCase` properties mapped to `snake_case` columns.

### Error handling
- Server uses oRPC and Elysia; throw `ORPCError` for expected API errors.
- Use explicit status codes for HTTP handlers.
- Avoid swallowing errors; log or rethrow with context.

### React/UI
- Utility class merging uses `cn` (`clsx` + `tailwind-merge`).
- Prefer functional components and hooks.
- Keep router types generated by TanStack Router (`routeTree.gen.ts`) untouched.

### Database/ORM
- Drizzle schema in `packages/db/src/schema`.
- Use `inferSelect` types for DB rows.
- Migrations output to `apps/server/migrations`.

### Generated files
- Do not edit `routeTree.gen.ts` directly; update routes and regenerate.

## Agent workflow tips
- Prefer editing in the smallest scope; follow existing patterns in touched files.
- Run `bun run check` after changes that touch TS/TSX or Tailwind classes.
- When updating API types, ensure web and app-ui consumers remain in sync.
