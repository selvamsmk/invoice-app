# Web Application

Modern invoice management web application built with React, Vite, and TanStack ecosystem.

## Overview

The web app provides a browser-based interface for the invoice management system. It uses the shared `@invoice-app/app-ui` package for UI components and routing, keeping web-specific code minimal.

## Tech Stack

- **Framework**: React 19
- **Build Tool**: Vite
- **Routing**: TanStack Router (via `@invoice-app/app-ui`)
- **State Management**: TanStack Query (via `@invoice-app/app-ui`)
- **UI Components**: shadcn/ui (via `@invoice-app/app-ui`)
- **Styling**: TailwindCSS
- **API Client**: oRPC (type-safe API)
- **Authentication**: Better-Auth

## Project Structure

```
apps/web/
├── src/
│   ├── main.tsx              # React entry point
│   ├── index.css             # Global styles
│   └── utils/
│       └── orpc.ts          # oRPC client setup
├── index.html                # HTML entry point
├── vite.config.ts           # Vite configuration
├── .env                     # Environment variables (not in git)
└── .env.example            # Environment template
```

**All routing, components, and business logic live in `@invoice-app/app-ui`**. This app is just a thin entry point that:
1. Sets up the oRPC client with the backend URL
2. Initializes the router and query client
3. Renders the shared App component

## Environment Variables

Create a `.env` file in `apps/web/` with:

```env
# Backend server URL
VITE_SERVER_URL=http://localhost:3000
```

For production, point `VITE_SERVER_URL` to your deployed backend URL.

## Development

From the monorepo root:

```bash
# Run web app in development mode
bun run dev:web

# Or run web + server together
bun run dev
```

The dev server runs on `http://localhost:5173`.

## Build

```bash
# Build for production
bun run build

# Build outputs to apps/web/dist/
```

The build creates an optimized static site ready for deployment.

## Deployment

The web app is a static React SPA that can be deployed to any static hosting:

### Vercel
```bash
# From apps/web/
vercel
```

### Netlify
```bash
# From apps/web/
netlify deploy --prod
```

### Azure Static Web Apps
```bash
# Using Azure CLI
az staticwebapp create ...
```

### Self-hosted (Nginx/Apache)
Build the app and serve the `dist/` directory.

**Important**: Ensure your server is configured for SPA routing (all routes should serve `index.html`).

## API Connection

The web app connects to the backend via oRPC:

```typescript
// apps/web/src/utils/orpc.ts
const baseUrl = import.meta.env.VITE_SERVER_URL ?? window.location.origin;
export const { queryClient, client, orpc } = createOrpcClient(baseUrl);
```

In production, set `VITE_SERVER_URL` to your backend API URL or deploy the backend on the same origin.

## Authentication Flow

1. User signs up/signs in via Better-Auth forms
2. Session cookie stored by browser
3. oRPC client automatically sends cookies with requests
4. Backend validates session on protected routes

See `@invoice-app/auth` for authentication configuration.

## Shared UI Package

This app uses `@invoice-app/app-ui` for:
- All routes and page components
- UI component library (buttons, forms, dialogs)
- TanStack Router setup
- TanStack Query hooks
- Authentication client

Web-specific code is minimal by design.

## Type Safety

- Full end-to-end type safety via oRPC
- Route types auto-generated by TanStack Router
- Server types automatically available in client code

## Performance

- Code splitting via Vite
- Lazy route loading via TanStack Router
- React Query caching for API responses
- Optimized production builds with tree-shaking

## Browser Support

Targets modern browsers (ES2020+). Update `vite.config.ts` build target if wider support needed.

## Troubleshooting

### Build Errors
```bash
# Clean and rebuild
rm -rf dist node_modules
bun install
bun run build
```

### API Connection Issues
- Check `VITE_SERVER_URL` is set correctly
- Verify backend server is running
- Check browser console for CORS errors

### Routing Issues on Deployment
Ensure your hosting provider is configured for SPA routing (all routes serve `index.html`).

## Development Tips

- Use React DevTools for component debugging
- Use TanStack Query DevTools (built into app-ui)
- Check Network tab for API request/response inspection
- Console errors in catch blocks are intentional (production error logging)
